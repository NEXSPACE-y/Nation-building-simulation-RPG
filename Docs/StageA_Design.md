# Stage A 設計固定

## 1. 適用範囲

本設計はStage Aの純粋ロジック検証だけに適用する。Stage B（Godot＋C#）およびStage C（Unreal Engine＋C++）の検討・実装は含まない。

正本にない国家名、NPC名、会話、人物設定、数値初期値は`fixture_only: true`のテストフィクスチャとして隔離する。本番データとして再利用しない。

## 2. 構成

```text
JSON fixture
  ↓ 厳格なスキーマ／件数／状態検証
Simulation（エンジン非依存C++23）
  ├─ 決定論的イベントキュー
  ├─ 目撃・伝聞・AI判断
  ├─ 会話・行動・関係・国家効果
  ├─ オフライン集約
  ├─ 原子的セーブ／ロード
  └─ 監査ログ・因果ログ
       ├─ CLI
       └─ CTest受入テスト
```

## 3. データモデル

| 正本 | Stage A |
|---|---|
| Player | `PlayerState`。所在地、資金、評判、犯罪記録、実績タグ、行動履歴を保持 |
| AI NPC | `AiNpcState`。20人、個別ID・役割・位置・現在状態・評価・関係・既知イベントを保持 |
| AI NPC State Definition | 各NPCのJSON内に状態1〜255を明示。状態6〜255はNPCごとの`UNDEFINED` |
| Transition Rule | トリガー、対象イベント、知覚種別、信用度範囲、優先度、遷移先、評価・関係効果をJSON定義 |
| NON AI NPC | 20人。所在地、シード、仮プロフィール、一時事件記憶を保持 |
| World State | 固定可能な`world_seed`、世界時刻、最終シミュレーション時刻、イベント履歴を保持 |
| Country State | `country_001`をマップ可能な独立構造として保持 |

状態表そのものは共有しない。共通化するのはローダー、条件評価器、イベント処理器、ログ形式、汎用アクション処理だけである。

## 4. Stage Aフィクスチャの仮値

ユーザー指定値に加え、次を検証専用値として明示する。

| 項目 | 仮値 |
|---|---:|
| `world_seed`既定値 | `424242` |
| 国家パラメータ初期値 | 原則`50`、`crime_level`のみ`10` |
| 目撃成立しきい値 | `0.35` |
| イベント連鎖上限 | 1ルートあたり`128`イベント |
| 使用状態 | 各NPCの1〜5 |
| 未使用状態 | 各NPCの6〜255を`UNDEFINED` |
| 関係値 | 上下限を設けない符号付き整数（Stage Aの因果確認用） |

目撃スコアは、AI NPCの注意度、場所の可視度、簡易遮蔽、事件の目立ちやすさから算出する。高度な物理視線判定は行わない。

## 5. イベント処理順

1. プレイヤー入力または時間入力からルートイベントを生成する。
2. NON AI NPC対象事件を一時記憶へ保存する。
3. 同一場所、注意度、可視度、遮蔽、事件顕著度から目撃者を決定する。
4. 噂の場合は`RUMOR_CREATED`→`RUMOR_TRANSFERRED`→`AI_NPC_HEARD_EVENT`を生成する。
5. AI NPCの現在状態に属する遷移ルールだけを評価する。
6. 優先度最大のルールを選び、同点時だけ決定論的ハッシュを使用する。
7. 状態、プレイヤー評価、関係値を変更する。
8. 会話と行動を別々に選択し、開始・完了イベントを生成する。
9. 報告先AI NPCまたは許可済み国家効果へ伝播する。
10. すべてのイベントへ`source_event_id`と`root_event_id`を設定する。

NON AI NPCの事件から国家状態を直接変更する経路は存在しない。国家変更は`CHANGE_COUNTRY_STATE`候補を選択したAI NPCの完了行動からだけ生成する。

## 6. 再現性

ルール同点時の値は、以下を連結して64-bit FNV-1aで生成する。

```text
world_seed | npc_id | event_id | current_state_id | simulation_tick
```

システム時刻は乱数へ使用しない。イベントIDは保存対象の単調増加シーケンスで生成する。JSONオブジェクトはキー順が安定する表現でスナップショット化する。

## 7. 時間とオフライン処理

- 現実60秒をゲーム60分へ変換する。
- 1秒単位の逐次シミュレーションは行わず、時間差を整数演算で集約する。
- 現実時間7日を超える入力は604,800秒へ切り詰める。
- `TIME_ELAPSED`と`OFFLINE_SIMULATION_COMPLETED`を因果接続して記録する。
- テストでは経過秒数を明示注入し、実行時刻に依存させない。

## 8. セーブ／ロード

セーブには、プレイヤー、20人のAI NPC、関係、既知イベント、会話・ルール使用履歴、20人のNON AI NPCの一時記憶、国家、世界時刻、イベント履歴、未処理イベント、イベント採番、連鎖処理数、スキーマ／シミュレーション版を含める。

一時ファイルへ完全書き込み後、既存正常保存をバックアップして置換する。置換失敗時は直前の保存を復元する。フィクスチャ、スキーマ、シミュレーション版の不一致はロードエラーにする。

## 9. 監査性

監査JSON Linesは、世界時刻、tick、イベント、ルート、`event_actor_id`（報告者・入力イベント主体）、`decision_npc_id`（判断・状態遷移主体）、`target_id`、AI NPC現在状態、成立ルール、選択ルール、遷移先、会話、行動、生成イベント、国家変更、決定論値を保持する。

因果JSON Linesは、全イベントの`event_id`、`event_type`、`source_event_id`、`root_event_id`、actor、targetを保持する。CLIの`trace`は任意の結果イベントから原因イベントまで逆参照し、時系列で表示する。

## 10. テスト構成

正本Scenario A〜Gを個別テストとして実行する。さらに、20×255状態契約、連鎖上限記録、指定プレイヤー行動の配線を検証する。Scenario Gはイベントキュー処理途中の保存を完了状態で上書きせず証拠として保持し、連続実行と保存・ロード・再開実行の正規化スナップショットについてSHA-256が一致することを検査する。
