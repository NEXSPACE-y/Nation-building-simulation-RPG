# 国家運営シミュレーションRPG
# 王都・2.5Dビジュアル仕様書 v0.1

---

## 0. 本書の位置付け

本書は、承認済みの「世界観基礎仕様書 v0.1」を前提として、最初に制作する王国の王都・王都周辺と、2.5D人物・牙鼠のビジュアル方針を固定する正本である。

本書で固定する対象は以下とする。

- 王都の基本的な見た目
- 王都を構成する7区画
- 限定的な魔法文明の見せ方
- 2.5D人物の画風と表示方向
- 人物アニメーションの基本契約
- 最初の魔物「牙鼠」の見た目と基本動作
- 3D背景と2.5Dキャラクターの責務境界
- Stage G-0描画PoCで検証するビジュアル範囲

本書は地理、戦闘、冒険サイクル、魔法体系、国家制度の詳細を確定するものではない。

本書にない本番設定、国家名、人物名、紋章、歴史、魔法体系を独断で追加してはならない。

# Part I：全体ビジュアル方針

## 1. 採用方向

採用する方向は以下で固定する。

```text
現実寄りの中世ヨーロッパ風
＋
王都の一部施設だけに見える限定的な魔法文明
＋
3D背景・地形・建物
＋
2.5D人物・モンスター
＋
斜め見下ろしカメラ
```

王都全体を魔法都市にはしない。

庶民の生活、住宅、工房、街路、農地、街道は、原則として通常の中世的な技術と生活様式を基礎とする。

魔法の恩恵は、王権、上位階級、専門施設、許可された組織へ偏って見える構成とする。

## 2. 表現上の重点

本作では、豪華さよりも以下を優先する。

```text
- 人が生活していること
- 階級差が画面から読み取れること
- 各区画の役割が一目で分かること
- NPCの位置と行動を把握しやすいこと
- 事件、報告、逃走、追跡の流れを追いやすいこと
- 2.5D人物が3D背景へ埋もれないこと
```

王都は清潔で均質なテーマパーク型都市にはしない。

区画ごとに舗装、建物密度、道幅、照明、汚れ、生活物、警備密度を変える。

# Part II：王都7区画

## 3. 区画一覧

王都と王都周辺は、以下の7区画で構成する。

```text
1. 王城・王宮周辺
2. 市場区画
3. 住宅区画
4. 貴族区画
5. 城門・街道
6. 下町・工房区
7. 郊外・農地・森
```

「王都全景」は区画ではなく、7区画の関係を確認するための俯瞰資料として扱う。

## 4. 王城・王宮周辺

### 4.1 役割

- 王権の中心
- 行政、儀礼、警備の中心
- 上位階級と許可者だけが自由に利用できる区画
- 王都における限定的な魔法文明が最も見えやすい区画

### 4.2 見た目

```text
- 高い石造建築
- 塔、城壁、門、広場
- 左右対称を意識した構成
- 整備された石畳
- 旗、装飾、衛兵詰所
- 庶民区画より広い道
- 視界を遮らない儀礼空間
```

### 4.3 魔法表現

使用してよい代表表現：

```text
- 許可制の魔法灯
- 出入口の簡易結界表示
- 治療または鑑定を行う専門設備
- 王権または公的機関の管理を示す魔法紋様
```

これらは世界中で普及した日用品として描写しない。

---

## 5. 市場区画

### 5.1 役割

- 商業の中心
- 商人、旅人、衛兵、住民が交差する区画
- Stage Eまでに実装したBROKER、GUARD、CAPTAINの反応を視覚化しやすい代表区画

### 5.2 見た目

```text
- 屋台
- 木造または木骨建築
- 荷車
- 商品箱
- 布製の日除け
- 看板
- 人の流れを妨げない中央通路
- 狭い脇道
- 商会または倉庫への導線
```

整然としすぎず、商品の搬入、値段交渉、警備、噂の流通が見える生活感を持たせる。

---

## 6. 住宅区画

### 6.1 役割

- 一般住民の生活圏
- プレイヤーが何もしない場合にも日常が継続していることを示す区画
- 近隣関係、評判、噂、生活事件を視覚化する区画

### 6.2 見た目

```text
- 木骨住宅
- 小さな庭
- 井戸
- 洗濯物
- 共同空間
- 路地
- 子どもや住民の生活物
- 補修跡のある建物
```

建物は均一にせず、世帯の経済差が外観に表れる構成とする。

---

## 7. 貴族区画

### 7.1 役割

- 上級階級の居住区
- 王城と庶民区画の中間
- 権力、資産、警備、閉鎖性を示す区画

### 7.2 見た目

```text
- 大型石造邸宅
- 庭園
- 塀
- 門
- 私兵または衛兵
- 広い道路
- 少ない露店
- 高品質な舗装
```

魔法設備は存在してよいが、王城より小規模かつ私有的なものとする。

## 8. 城門・街道

### 8.1 役割

- 王都内外を接続する区画
- 入国、出国、検問、物流、旅人、冒険家の交差点
- 王都周辺の危険と都市生活を接続する境界

### 8.2 見た目

```text
- 大型城門
- 城壁
- 検問所
- 衛兵詰所
- 荷車待機場所
- 宿または馬屋
- 石畳から土道へ変化する境界
- 街道標識
```

都市側と郊外側の照明、舗装、密度が明確に変わること。

## 9. 下町・工房区

### 9.1 役割

- 職人、日雇い、運搬、修理、製造の中心
- 王都の生活を支える労働区画
- 火災、事故、価格、資源不足などの事件が発生しやすい区画

### 9.2 見た目

```text
- 工房
- 炉
- 倉庫
- 木材、金属、布、樽
- 狭い路地
- 煙
- 作業音を想起させる設備
- 建物の増改築跡
```

魔法は一般化させず、ごく一部の認可工房または専門職だけが利用している程度とする。

## 10. 郊外・農地・森

### 10.1 役割

- 王都の食料と資源を支える区画
- 街道、農地、牧草地、林縁を含む
- 最初の魔物「牙鼠」が出現する検証地域

### 10.2 見た目

```text
- 農地
- 柵
- 納屋
- 小屋
- 牧草地
- 林
- 街道
- 水路または小川
- 王都を遠景に確認できる場所
```

王都から離れるほど人工物と警備密度を減らす。

# Part III：限定的な魔法文明

## 11. 見せ方の原則

魔法は一部の人間だけが使用できる。

そのため、魔法表現は「誰でも使える便利な電化製品」のように配置してはならない。

```text
一般生活：
火、油灯、手作業、家畜、荷車、職人技術

限定施設：
魔法灯、結界、治療、鑑定、許可制設備
```

魔法設備には、管理者、利用資格、費用、権力差が存在することを画面から推測できる構成とする。

## 12. 禁止する見せ方

```text
- 全街路へ均等に配置された魔法灯
- 庶民住宅での一般的な魔法家電
- 誰でも自由に使う転移設備
- 王都全体を覆う過剰な光学演出
- 魔法だけで生活が完結する都市
- 世界観基礎仕様書にない魔法体系の独断設定
```

# Part IV：2.5D人物仕様

## 13. 基本画風

人物は以下で固定する。

```text
頭身：
5.5〜6.5頭身

輪郭：
やや強め

陰影：
小表示で潰れない中程度
描き込みすぎない

色：
3D背景に埋もれない
役割と所属を識別しやすい

表現：
写実一辺倒ではなく、ゲーム画面での判読性を優先
```

顔の細部よりも、髪型、衣装、装備、体格、姿勢によって個体と役割を判別できることを優先する。

## 14. 方向

人物は8方向を基準とする。

```text
正面
右前
右
右後
背面
左後
左
左前
```

左右反転だけに依存せず、装備、紋章、利き手などの破綻が発生しない構造を目標とする。

Stage G-0では技術検証のため左右反転を利用してもよいが、本番採用可否を証拠へ明記する。

## 15. 基本アニメーション

### 15.1 人物共通

```text
IDLE       待機
WALK       歩行
TALK       会話
```

### 15.2 Stage E行動の表示候補

```text
WARN
REPORT
INVESTIGATE
PROTECT
ARREST
FLEE
REFUSE_TRADE
HELP
```

Stage G-0ですべての専用作画を完成させる必要はない。

Stage G-0では最低限、以下を実装する。

```text
IDLE
WALK
TALK
WARN
REPORT
FLEE
```

`ARREST`は簡易専用モーションまたは明示的な暫定表示で検証する。

## 16. フレーム数の基準

以下は初期制作基準であり、必要に応じて同一アニメーション内で調整できる。

```text
IDLE：1〜4フレーム
WALK：6〜8フレーム
TALK：2〜4フレーム
WARN：2〜4フレーム
REPORT：2〜4フレーム
ARREST：4〜8フレーム
FLEE：WALKを高速再生してよい
```

## 17. スプライト共通契約

```text
- 足元を全フレームで同じ基準位置へ合わせる
- pivotは足元中央
- 方向間で身長と接地位置を変えない
- フレーム間で不要な拡大縮小を行わない
- 透明余白を過剰に持たせない
- 画像の見た目と衝突判定を分離する
- Actorの論理座標はスプライト画像に依存させない
```

## 18. 外見差分

将来の量産では、以下の4系統を差し替え可能にすることを目標とする。

```text
身体・顔
髪
衣装
装備
```

ただしStage G-0では、合成レイヤー量産機構を完成させない。

Stage G-0では、プレイヤー、GUARD、BROKER、CAPTAIN、一般住民の代表見本を用いた表示方式の成立確認を優先する。

# Part V：牙鼠仕様

## 19. 基本定義

名称：

```text
牙鼠
```

読み：

```text
がそ
```

牙鼠は、王都周辺に生息する大型の鼠型危険生物である。

悪や闇から生まれた存在ではない。

## 20. 生態上の方向

```text
- 下水道、廃屋、農地、林縁に生息
- 群れで行動する可能性がある
- 小動物、作物、保存食、家畜飼料を狙う
- 繁殖と餌不足により人里へ接近する
- 鋭い前歯と素早い移動を持つ
- 成長個体は中型犬に近い大きさまで大型化し得る
```

詳細な繁殖率、戦闘能力、素材価値は後工程で定義する。

## 21. 見た目

```text
- 黒または暗灰色を基調とする粗い体毛
- 通常の鼠より明確に大型
- 前歯が目立つ
- 低い姿勢
- 長い尾
- 俊敏さを感じる体型
- 過度な悪魔化や装飾を行わない
```

## 22. 方向と動作

牙鼠も2.5Dの8方向表示を基準とする。

Stage G-0で必要な動作：

```text
IDLE        待機
MOVE        移動
CHARGE      突進
BITE        噛みつき
HIT         被弾・よろけ
DEATH       死亡
```

戦闘システム未確定のため、Stage G-0ではダメージ計算や戦利品処理を実装しない。

動作は、表示確認用fixtureまたはデバッグ入力から再生してよい。

## 23. 牙鼠のゲーム内役割候補

```text
- 初心者向け危険生物
- 農地被害
- 倉庫被害
- 街道または下水道の問題
- 衛兵、農民、商人、冒険家の反応対象
- 素材採取対象
- 数の増減による地域問題
```

上記は世界観上の役割候補であり、Stage G-0でシステム実装しない。

# Part VI：3D背景と2.5Dキャラクターの境界

## 24. 3D側の責務

```text
- 地面
- 地形
- 建物
- 壁
- 小物
- 植物
- 光源
- 天候
- NavMesh
- 衝突形状
- 高低差
- 遮蔽
```

## 25. 2.5D側の責務

```text
- 人物と牙鼠の見た目
- 方向別表示
- アニメーション
- 視認性
- 足元基準
- 表示用簡易影
```

## 26. 因果コアとの境界

2.5Dキャラクター側へ判断ロジックを置いてはならない。

```text
因果コア：
判断、状態遷移、行動決定、関係、国家作用

Unreal表示層：
位置、向き、表示アニメーション、エフェクト、UI
```

表示層は、コアが決定した行動を表現する。
